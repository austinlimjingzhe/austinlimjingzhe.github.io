A

AutoML Application

An attempt at creating a no-code AutoML application using R shiny, caret and iml where users can upload their own datasets, select their desired target variable and train multiple machine learning models. The application also features explanable AI and allows users upload a new dataset to make new predictions off the trained models.

This application was developed in tandem with the Data Preprocessing Application.

  • Tools used: R, shiny, caret, iml
  • Category: Machine Learning, Web Application, Explainable AI
  • Year: Jul 2022

C

Cold Call Analysis

This project was a school project for the module 'Statistical Analysis with R'.

Using data from an anonymous American bank sourced from a data mining competition, we used statistical models to predict the success of cold calls for a car insurance campaign. From our most accurate Generalized Linear Models, we provided recommendations for the bank to improve their cold call success rates.

  • Tools used: R, tree, glmnet, leaps
  • Category: Machine Learning, Statistics
  • Year: Apr 2020

D

Data Preprocessing Application

An attempt at creating a no-code datapreprocessing application using R shiny, DateExplorer, missingForest, smotefamily and more, where users can upload their own datasets and clean their data using 10 different features. These include advanced imputation methods, dimensionality reduction, handling imbalanced classes and rolling window transformations on top of standard data wrangling and transformation techniques.

This application was developed in tandem with the AutoML Application.

  • Tools used: R, shiny, missingForest, Amelia, smotefamily
  • Category: Data Cleaning, Web Application
  • Year: Jul 2022

Exchange Rate Policy As A Monetary Policy Instruments

This project was a school project for the module 'Financial Markets and Monetary Policy' in collaboration with the State Bank of Vietnam (SBV).

We analyze the economic environment of Vietnam, existing measures of the SBV and case studies from other countries using exchange rate as a monetary policy. Finally, we attempt to support our argument that an exchange rate policy is an effective channel for monetary policy using a Vector Autoregressive (VAR) model in R.

  • Tools used: R, vars, tseries
  • Category: Time Series Analysis
  • Year: Apr 2021

S

Shopee Code League 2020

Shopee Code League is a 2-month online coding challenge consisting of a series of competitions designed by Shopee tech teams where participants must analyse the dataset, draw insightful conclusions and solve the problems related to the E-commerce platform's day-to-day operations.

My group took part in the Marketing Analysis challenge where we built a model that can predict whether a user opens the emails sent by Shopee to evaluate the effectiveness of their marketing campaigns. We also took part in the Sentiment Analysis challenge where built a model that can classify product reviews so that Shopee can understand their customers needs and quickly adapt their services to provide a much better experience.

  • Tools used: Python, keras, ktrain
  • Category: Deep Learning, BERT
  • Year: Aug 2020

Stock Prediction Analysis

As information about a company can affect its stock price, this project combines the Google Trends data onto a Long-Short Term Memory (LSTM) to conduct time series analysis on the stock price of Apple (AAPL). I investigate the hypothesis that in the short term, the model that incorporates Google Trend would be more accurate as models using a longer time period could be using outdated sentiments and the results seem to be in line with this expectation.

  • Tools used: Python, yfinance, keras, pytrends
  • Category: Deep Learning, Time Series Analysis
  • Year: Oct 2021

Weather Forecasting in Singapore: Predicting the Occurrence of Rainfall Using Classification Techniques on Scraped Meteorological Data

This project was a school project for the module 'Data Mining and Business Analytics'.

We aim to use information available via Singapore's National Environment Agency's public APIs on data.gov.sg (temperature, humidity, wind) to predict the presence or absence of rain. An SQL database is used to store the data obtained from the APIs.

  • Tools used: Python, SQL, sklearn, imblearn, geopandas
  • Category: Python, Data Mining, Imbalanced Classification, API, Geospatial Data Analysis
  • Year: Apr 2022
